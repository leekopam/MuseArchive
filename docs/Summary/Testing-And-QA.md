## 6. 수동 QA 체크리스트

## 6.1 홈

- 컬렉션/위시리스트 전환이 정상 동기화되는가
- 검색창 열기/닫기가 자연스러운가
- 정렬 옵션이 기대 순서로 반영되는가
- 재정렬 후 앱 재실행 시 순서가 유지되는가
- 길게 누른 카드 액션 시트가 올바르게 동작하는가

## 6.2 추가/수정

- 제목/아티스트 입력 후 자동 저장되는가
- 뒤로 가기 직전 마지막 수정이 반영되는가
- Discogs/VocaDB/MusicBrainz 검색 결과 병합이 정상인가
- Spotify 설정이 없을 때 사용자가 이해 가능한 메시지가 나오는가
- 이미지 선택 후 다시 열었을 때 커버가 유지되는가
- 여러 디스크 트랙 편집이 꼬이지 않는가

## 6.3 상세

- 상단 액션이 모든 스크롤 위치에서 읽기 쉬운가
- 즐겨찾기 토글 직후 홈 상태와 일치하는가
- 편집 후 상세에 최신 정보가 반영되는가
- 삭제 후 이전 화면 갱신이 되는가
- 링크 버튼이 없는 앨범에서 버튼이 숨겨지는가

## 6.4 아티스트

- 이미지 추가/삭제가 반영되는가
- alias 검색이 홈/곡 목록에 반영되는가
- 발매일 정렬 토글이 바로 목록에 반영되는가
- 원본 앨범 강조 표시가 자연스러운가

## 6.5 곡 목록

- 헤더 트랙이 제외되고 실제 곡만 나오는가
- alias로 검색했을 때 관련 곡이 보이는가
- 위시리스트 포함 토글이 즉시 반영되는가
- 동일 곡의 다른 앨범 찾기가 정확한가

## 6.6 설정

- 테마 전환이 앱 재실행 후에도 유지되는가
- Discogs/Spotify 연결 테스트 결과가 정확한가
- 백업 파일 저장이 되는가
- 복원 실패 시 사용자에게 명확히 보이는가
- 업데이트 확인이 네트워크 오류 시도에서도 안전한가

## 6.7 플랫폼별 수동 점검

### Android

- release 빌드/설치 가능 여부
- 백업 파일 저장 권한 동작
- APK 업데이트 흐름

### iOS

- 카메라 권한 요청
- 사진 접근 권한 요청
- 바코드 스캔/이미지 선택 가능 여부

### macOS

- 네트워크 호출 허용 여부
- 파일 저장/열기 entitlement 문제 유무

## 7. 제안하는 테스트 확장 순서

3. 홈 정렬/재정렬 테스트
4. 상세 화면 액션 테스트
5. 설정 화면 실패 경로 테스트
6. 플랫폼별 수동 QA 체크시트 정착

## 8. 테스트 관점 최종 판단

현재 테스트는 "앱이 완전히 깨지지 않았는지" 정도의 얇은 기준선은 제공한다.  
하지만 이 앱의 핵심 가치인 다음 영역은 아직 자동 검증이 부족하다.

- 로컬 데이터 보존
- 자동 저장 편집 흐름
- 백업/복원 안정성
- 홈 탐색과 재정렬
- 설정 기반 외부 연동

따라서 앞으로의 수정 작업은 가능한 한 "테스트 추가와 함께" 진행하는 것이 안전하다.
